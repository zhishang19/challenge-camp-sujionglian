# 多源记忆训练数据说明（学生版）

本数据包围绕麒麟 OS Agent 记忆优化赛题构建，覆盖多源数据接入与整合、偏好记忆动态捕捉、知识记忆提取与整合、记忆流转、量化评测机制五类能力。

## 难度梯度

| 阶段 | 训练重点 | 预期能力 |
|------|----------|----------|
| D1 | 观察 demo | 能提出清洗假设，区分删除、保留、复查 |
| D2 | 入门合并与去噪 | 能完成字段统一、基础去重、文本规整 |
| D3 | 基础验收 | 能处理空值、重复、时间规整与敏感信息 |
| D4 | 多源清洗实战 | 能把对话、知识、偏好、配置、工具结果整理成可用记忆材料 |
| D5 | 记忆冲突与流转 | 能判断短期、长期、偏好、知识之间的保留、覆盖、遗忘关系 |
| D6 | 评测与验收 | 能用清洗结果支撑检索、回答、入库和质量评估 |

`generated/` 目录是批量数据，每天约 1 万条。它会包含比 demo 更丰富的场景、噪声和异常组合，适合训练脚本性能、稳定性和质量统计。

## 清洗目标

不要急着写规则。先观察每份材料中哪些内容可靠、哪些内容应被丢弃、哪些内容要延后判断，再设计处理流程，最后能呈现出你清洗后的效果（表格或者报告分析都可以）。

### D1 预期观察效果

- 能说出哪些内容可能是有效语义，哪些内容可能是噪声。
- 能发现“临时要求”和“长期偏好”不一定相同。
- 能意识到隐私内容不能直接进入长期记忆。

### D2 预期清理效果

- 同一用户的重复行为被合并。
- 时间能排序，无法确认的时间被明确标记。
- 工具结果中的执行状态、耗时和有效输出可直接读取。
- 停顿词、装饰符、无意义缓存标记被移除，但不改变原意。

### D3 预期清理效果

- 空消息、重复消息、缺失用户标识等问题被识别。
- 敏感信息不再以原文出现。
- 对话仍保留原始先后关系。
- 每条有效内容可追溯到原会话。

### D4 预期清理效果

- 对话内容变得简洁、可检索、可入库。
- 偏好类记忆能区分“默认设置”“用户明确表达”“用户后来纠正”。
- 知识类内容能形成标题、步骤、标签、注意事项等稳定结构。
- 工具调用结果保留关键结果，删除日志噪声和无意义口语。
- 冲突、遗忘、敏感信息、错别字被单独记录，便于人工复查。

### D5 预期清理效果

- 同一用户的最新偏好能覆盖旧偏好，但历史变化可追踪。
- 临时指令不会错误写入长期记忆。
- 明确要求“不要记”“忘记”的内容不会进入可检索记忆。
- 同义说法、错别字和大小写差异不会造成重复记忆。
- 冲突项能输出原因、证据和建议处理动作。

### D6 预期清理效果

- 检索问题能命中正确知识、偏好或工具执行记录。
- 回答结果能体现用户偏好与安全规则。
- 评测报告能统计去重、脱敏、冲突、召回、误召回等关键指标。
- 失败样例能说明失败原因，而不是只给出分数。

## 提交建议

- 每个阶段输出一份 `report.md`，写清楚你发现的问题、采用的规则、保留的争议点。
- 清洗脚本应支持重复运行，重复运行不应产生重复数据。
- 不确定的内容不要强行删除，优先标记为 `needs_review`。
